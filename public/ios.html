Artigos para adapter do iOS

Essa coletânea possui diversos links úteis para você aprender a fazer a versão iOS do seu app (com Turbolinks).

Documentação oficial
by Turbolinks Team

Documentação muito bem escrita! Explica como funciona e como utilizar o adapter e vale a pena pelo menos rodar a demo feita por eles mostrando as features do adapter iOS.
Além da documentação, vale a pena ver o talk da RailsConf 2016 "Turbolinks 5: I Can’t Believe It’s Not Native!" em que Sam Stephenson explica muito bem a motivação do Turbolinks e também faz uma demo de como criar um App híbrido iOS. Para o link, cheque a nossa seção de Vídeos.
Para um ver o código de um app real, confira o próximo artigo!

Código fonte do MasterApp iOS
by The Master App Team

O Master App é um app para você ver e compartilhar receitas. O código disponibilizado mostra como criar uma navegação nativa usando Tab Bar. Além disso, também é possível ver como está sendo gerenciada a navegação no app, como é feito o gerenciamento de sessão do usuário e o CRUD de receitas. Tal projeto, assim como a Demo fornecida na documentação do Turbolinks-iOS, são ótimos pontos de partida para você iniciar sua aplicação.

Tab Bar native-hybrid app with Xcode storyboards and Turbolinks 5
by Michiel Sikkes

Para saber como setar uma navegação com tabbar passo a passo. Ponto de atenção: nesse tutorial ele mostra como ter 2 tabs no seu app, porém a solução proposta tem um problema caso você queira compartilhar estado entre as duas tabs (por exemplo, informar que o usuário está logado). Para isso, lembre-se de compartilhar a WKProcessPool entre todas instâncias de WebView do App. Para mais informações, olhe o código fonte do MasterApp.

Building an iOS app with Turbolinks-iOS
by Joshua Jansen

Para saber mais sobre forms e como fazer a comunicação servidor rails com o adapter iOS.

DeviceToken and turbolinks-ios
by Michiel Sikkes

Sobre enviar device token (para push notifications) a partir do adapter iOS. Tal técnica também pode ser utilizada para enviar um authToken da sessão do usuário. Um outro approach seria enviar o token como parâmetro da URL (atualmente utilizamos essa técnica no MasterApp para enviar locale e o authToken). Para mais informações sobre assunto, vale também conferir a seguinte issue no próprio repositório do turbolinks-ios.
